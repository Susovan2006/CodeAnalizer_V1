  • Spring MVC is supported in PCF.
  • Tomcat Apps are best fit for PCF cloud.
  • If the buildpack doesn't support WebLogic, minor refactor is neeed to move it to Tomcat.
  • Should be compatable with PCF/TAS.
  • REST API is a good fit in PCF Container.
  • Spring Batch can run in PCF, but Auto scaling might not be possible.
  • XLS File generation is supported in PCF, need to find if any NAS is being used, Any Encryption at Rest is in Place and the size of the XLS.
  • Email Feature is available in PCF, but it will use a different SMTP host.

  • As the Application will be running in container, we can't have the JNDI config in the servers, It has to be migrated to Hikari Connection Pool'.

  • If the application has Cron/Quartz, Auto scaling is not possible. There has to be some rewrite to segrigate the Batch Portion.
  • If the application is Writing/Reading files in the Physical server, in PCF it should be NAS/HCP etc, So a portion of refactor has to be done.
  • PDF Generation is supported in PCF, but need to know the size of the PDF, the PDF generation consumes huge memory, so rewrite might be required to split into small modules.
  • ehcache Cache is a not a good fit for PCF/TAS, so it's better to move it to centralized Cache like Redis'.
  • Siteminder is supported in PCF, but it's recommended to use PingFed. but this is based on Org.'
