Suggestion on General

  • In PCF application runs in a container, so SpringBoot is the best suit for PCF Environment, the cloud version of the code will use SpringBoot as well.
  • Tomcat Apps are best fit for PCF cloud.
  • REST API is a good fit in PCF Container.
  • Email Feature is available in PCF, but it will use a different SMTP host.
  • Spring Security is supported in PCF, but it's recommended to use PingFed. but this is based on Org.
  • Junit 5 is supported in PCF with buildpack java 17.
  • Maven Project is supported in PCF.
  • Java 1.8 is supported in PCF, as long as the buildpack is available in ORG.
  • slf4j is supported in PCF.
  • Logback Connector is supported in PCF.

Suggestion on Rewrite

  • If the application has Cron/Quartz, Auto scaling is not possible. There has to be some rewrite to segrigate the Batch Portion.
  • If the application is Writing/Reading files in the Physical server, in PCF it should be NAS/HCP etc, So a portion of refactor has to be done.
  • AES/Base64 Encryption is supported in PCF, but the Org might not allow.
  • Server/IP/Socket specific logic will not support in PCF. here everything will be running in container.
  • need to change the packages from javax to jakarta as a part of java 17 upgrade..
  • hardcoded Password is a security risk, need to move them to some secure storage like hashicorp Vault.

Suggestion on Refactor

  • Plain Java Servlet Application can run in PCF, but needs refactor.

Suggestion on Risk

  • Database TX can be possible in PCF/TAS, need to make sure there is fault tolerance.
